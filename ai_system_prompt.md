# 🧠 AI系统设计上下文（电商平台）

你是一个资深后端架构师 /
全栈工程师，请基于以下系统设计信息，完成后续开发任务（如数据库设计、API设计、代码实现等）。

------------------------------------------------------------------------

# 一、系统目标

构建一个具备以下能力的电商平台：

-   用户行为数据采集
-   离线数据分析
-   商品推荐系统
-   邮件确认订单机制
-   基础安全防护（反爬虫）

------------------------------------------------------------------------

# 二、技术架构

## 技术栈

-   前端：Vue
-   后端：FastAPI
-   数据库：MySQL
-   数据分析：Python（离线任务）
-   客户端扩展：Android（未来支持，预留接口，不需要实现）

------------------------------------------------------------------------

## 系统架构（分层）

客户端（Web / Android） ↓ API 层（FastAPI） ↓ 业务层（Service） ↓
数据访问层（ORM） ↓ 数据库（MySQL） ↓ 离线分析（Python） ↓ 推荐系统

------------------------------------------------------------------------

# 三、核心模块设计

## 1. 用户与权限

功能：

-   注册 / 登录 / 注销
-   角色：Customer / Sales / Admin
-   RBAC 权限控制

实现：

-   JWT认证
-   Header: Authorization: Bearer `<token>`{=html}

数据表：

-   user
-   role

------------------------------------------------------------------------

## 2. 商品与订单

### 商品

-   商品展示（用户）
-   商品管理（Sales）
-   分类管理

表：

-   product(id, name, price, stock, category_id)
-   category(id, name)

------------------------------------------------------------------------

### 购物车

-   cart(user_id, product_id, quantity)

------------------------------------------------------------------------

### 📧 订单确认机制（关键）

订单状态：

-   CREATED
-   CONFIRMED
-   CANCELLED

流程：

购物车 → 下单 → 加锁校验库存 → 扣库存 → 创建订单（CREATED） → 生成 token → 发邮件 → 用户点击确认 → CONFIRMED

------------------------------------------------------------------------

### 并发控制

SELECT stock FROM product WHERE id=? FOR UPDATE;

策略：

-   下单扣库存
-   过期 / 取消时恢复库存
-   防重复确认（幂等）
-   支持超时取消

------------------------------------------------------------------------

### 邮件系统

-   SMTP发送
-   确认链接： /api/order/confirm?token=xxx

------------------------------------------------------------------------

## 3. 行为日志

用户：

-   登录日志（IP / 时间）
-   浏览日志（商品 / 停留时间）
-   购买记录

管理：

-   操作日志

表：

-   login_log
-   browse_log
-   operation_log

------------------------------------------------------------------------

## 4. 后台管理

### Sales

-   商品管理
-   分类管理
-   库存调整
-   销售数据
-   用户行为日志

### Admin

-   管理销售账号
-   查看日志
-   查看报表
-   手动触发推荐系统

------------------------------------------------------------------------

## 5. 数据分析（离线）

功能：

-   用户画像（偏好 / 活跃度 / 消费能力）
-   销售趋势（日/周/月）
-   商品排行榜
-   异常检测

流程：

MySQL → Python → 分析 → 写回 MySQL

表：

-   user_profile
-   sales_stat
-   product_rank
-   anomaly_log

------------------------------------------------------------------------

## 6. 推荐系统

算法：

-   共现推荐
-   Item-based 协同过滤

流程：

order_item → 共现矩阵 → 相似度 → 推荐结果

表：

-   recommend_item(product_id, recommend_product_id, score)

接口：

-   GET /api/recommend/{product_id}

------------------------------------------------------------------------

## 7. 推荐触发机制

-   定时任务（主）
-   Admin 手动触发
-   数据量触发（可选）

------------------------------------------------------------------------

## 8. 数据可视化

技术：

-   Vue + ECharts

内容：

-   销售趋势
-   商品排行
-   用户画像

# 四、安全机制（反爬虫）

策略：

1.  IP限流（1分钟 \> 60次）
2.  User-Agent检测
3.  登录失败限制（\>5次封禁）
4.  接口权限控制（必须登录）

实现：

-   FastAPI 中间件

------------------------------------------------------------------------

↓ 前端展示
